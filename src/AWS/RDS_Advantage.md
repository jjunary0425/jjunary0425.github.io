# RDS를 사용하면 얻을 수 있는 이점

RDS는 관리형 서비스
- 프로비저닝, OS 패치 자동화
- 특정 시점을 지속 적으로 백업, 리스토어
- 대시보드를 이용한 모니터링
- Read replica를 이용하여 읽기 성능 향상
- Mutil AZ로 재해 복구 가능
- 스케일링

## RDS 스토리지 오토 스케일릴

데이터베이스의 스토리지가 부족해 지고 있는 것을 감지하면 자동으로 스케이링

- 수동 스케일링 방지
- 최대 용량 임계치 적용 가능
- Worklads를 예측하기 힘든 경우 유용
- 모든 엔진(MariaDB, MySQL,Postgre ... )에서 지원
